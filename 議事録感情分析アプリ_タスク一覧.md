# 議事録感情分析アプリ タスク一覧（更新版）

## システム概要
音声ファイル（MP3/MP4）から文字起こし→Bedrock要約→感情分析を行い、結果をS3に保存する議事録分析アプリ

## 開発体制
- **エンジニア1**: インフラ担当（AWS環境構築）
- **エンジニア2**: フロントエンド担当（Web UI開発）  
- **エンジニア3**: 統合・テスト・ドキュメント担当

## 事前準備（開発開始前）
- [ ] AWSアカウント準備・管理者権限設定
- [ ] AWSコンソールアクセス確認
- [ ] テスト用MP3/MP4ファイル準備（2-3分の会議音声）
- [ ] Bedrock Claude 3 Haikuモデルアクセス申請・有効化
- [ ] GitHubリポジトリ作成・共有

## 緊急時の優先度調整

### 時間が足りない場合の削除候補
1. **MD形式ダウンロード** → TXT形式のみ
2. **Cognito認証** → 認証なしでの動作
3. **ドラッグ&ドロップ** → ファイル選択ボタンのみ
4. **エラーハンドリング詳細化** → 基本的なエラー表示のみ
5. **CSSアニメーション** → シンプルなスタイルのみ

### 時間に余裕がある場合の追加候補
1. **自動更新機能** → setIntervalで定期的な状態更新
2. **プログレスバー** → アップロード進捗の視覚化
3. **ファイルプレビュー** → アップロード前のファイル情報表示
4. **グラフ表示** → Chart.jsで感情分析結果のグラフ化
5. **ダークモード** → CSS変数でテーマ切り替え
